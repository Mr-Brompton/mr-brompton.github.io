The sorting problem

Matthew Brompton



Sorting is a great place to start when attempting to understand algorithms. The list of potential uses is huge, but a couple examples we might use every day are excel spreadsheets being used to sort by column and Google sorting results by ranking. Because sorting is such a fundamental step it is a good idea to have an efficient algorithm for it. That should be easy right? Not so much.

The sorting problem is almost as old as the computer itself. Some of the first algorithms were designed in 1951 and are still often used (rightly or wrongly) today, Bubble sort for example was first analysed in 1956 while a much more recent algorithm library sort was first published in 2006. There is no evidence as of yet that we have developed them all.

Just to get a quick idea of the different algorithms available to us, and how each one works, check out this video on sorting.

That video quickly runs through 15 sorting algorithms, showing exactly how each works. If we understood anything from it at this stage, we are certainly superheroes, and this blog isn’t for us. If however we are normal humans playing the video a normal speed – we would have seen a few flashing bars being slowly rearranged in height order. Riveting stuff!

All we need to know for now is that sorting isn’t so simple. Different algorithms have different advantages and disadvantages in different scenario’s, and it’s a good idea as a programmer to have a handle on a few.

This part of the algorithm series will go through a few of them, each time looking at a different aspect of algorithm design and analysis, so if we jump to a particular page and don’t understand something, it may be that we need to go back and read a few others.
